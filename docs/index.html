Release Shape

Reliable agents need runtime controls, not prompt folklore.

Each pattern in this repo turns an agent claim into a system decision: when to intervene, what evidence to require, and what state is allowed to survive promotion.

  • Decision-time guidance injects local rules only when the trajectory warrants it.
  • Self-verification reruns behavior from captured failure evidence before accepting output.
  • Reversible sandboxing isolates risky work and promotes only approved diffs.

Pattern Catalog

Three controls, one reliability shape.

Pattern 01

Decision-Time Guidance

Static prompts turn every reminder into a tax on every decision. This pattern classifies the current trajectory and injects a temporary guidance card only when a known failure shape appears.

  • Detects repeated errors, edit loops, unsafe commands, and fake-data escape.
  • Uses scores, card budgets, cooldowns, and evidence payloads.
  • Inspired by Replit's decision-time guidance engineering post.
Experiments: 3 / 3 pass Open pattern

Pattern 02

Self-Verification Loop

A generated app can look complete while failing the actual user outcome. This pattern separates shallow checks from behavioral verification and repairs from captured failure evidence.

  • Catches Potemkin success messages that do not persist state.
  • Runs executable scenarios and records UI, state, and log evidence.
  • Inspired by Replit's automated self-testing engineering post.
Experiments: 3 / 3 pass Open pattern

Pattern 03

Reversible Sandbox Execution

Agents need room to explore. Product systems need recovery, auditability, and promotion gates. This pattern treats agent work as a candidate overlay on top of a checkpoint.

  • Creates checkpoints and forked sandboxes.
  • Rejects destructive data changes and non-reversible external effects.
  • Inspired by Replit's snapshot engine engineering post.
Experiments: 4 / 4 pass Open pattern
