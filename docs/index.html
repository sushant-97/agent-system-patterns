Pattern 01 3 / 3 pass

Decision-Time Guidance

Watch the agent trajectory, detect repeated failure shapes, and inject a brief evidence-backed correction for the next decision only.

  • Stops prompt bloat from static reminder piles.
  • Uses scores, cooldowns, and card budgets.
  • Best for loops, unsafe moves, and fake progress.

Pattern 02 3 / 3 pass

Self-Verification Loop

Separate shallow completion checks from behavioral proof so the agent has to demonstrate working outcomes instead of persuasive output.

  • Catches Potemkin success that never changes state.
  • Uses runtime evidence to drive repair.
  • Best for UI, API, and tool-driven workflows.
Read detail Source README
Pattern 03 4 / 4 pass

Reversible Sandbox Execution

Let the agent attempt risky work inside a forked checkpoint, then promote only the subset of changes that pass verification and merge policy.

  • Keeps rollback power with the system.
  • Rejects destructive or irreversible side effects.
  • Best for stateful, high-blast-radius changes.

Chooser

Match symptom to control

This layer is original to the repo: a quick decision surface for readers who want to apply the patterns, not just read about them.

Symptom

The agent keeps circling the same failed path.

Start with Decision-Time Guidance. You need a local intervention before more work compounds the loop.

Symptom

The output looks done, but the behavior is unproven.

Start with Self-Verification Loop. You need executable evidence before accepting completion.

Symptom

The agent needs to touch state you cannot casually risk.

Start with Reversible Sandbox Execution. You need rollback and promotion gates around the attempt.
