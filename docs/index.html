Pattern 01 3 / 3 pass

Decision-Time Guidance

Use when the run starts drifting: repeated errors, unsafe cleanup, or fake progress. The controller injects one short correction for the next turn.

  • Boundary: next decision.
  • Evidence: trajectory signals.
  • Output: one guidance card.

Pattern 02 3 / 3 pass

Self-Verification Loop

Use when the artifact looks plausible but the intended behavior has not been proven. The verifier runs the scenario, captures evidence, and reruns after repair.

  • Boundary: acceptance.
  • Evidence: scenario run.
  • Output: pass/fail plus repair signal.
Read detail Source README
Pattern 03 4 / 4 pass

Reversible Sandbox Execution

Use when the agent needs to touch files, data, or side-effect boundaries. The controller forks state, verifies the candidate, and promotes approved diffs.

  • Boundary: durable state.
  • Evidence: sandbox diff.
  • Output: promote or reject.

Operating Map

Where the controls sit

These patterns sit around different moments in a run: before the next decision, before acceptance, and before durable promotion.

Input

User intent

Task, constraints, files, tools, and expected outcome enter the harness.

Control 01

Guide trajectory

Decision-time guidance fires only when the current behavior shows a known failure shape.

Attempt

Agent work

The agent edits, calls tools, runs commands, and produces a candidate result.

Control 02

Verify behavior

Self-verification checks the actual user outcome, captures evidence, and reruns after repair.

Control 03

Promote safely

Reversible sandboxing keeps risky work isolated until policy accepts the diff.

Chooser

Match symptom to control

This layer is original to the repo: a quick decision surface for readers who want to apply the patterns, not just read about them.

Symptom

The agent keeps circling the same failed path.

Start with Decision-Time Guidance. You need a local intervention before more work compounds the loop.

Symptom

The output looks done, but the behavior is unproven.

Start with Self-Verification Loop. You need executable evidence before accepting completion.

Symptom

The agent needs to touch state you cannot casually risk.

Start with Reversible Sandbox Execution. You need rollback and promotion gates around the attempt.
