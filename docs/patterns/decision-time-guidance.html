Decision-Time Guidance

Inject guidance only when the run needs it.

This pattern treats guidance as a runtime control layer. The agent does not carry every possible reminder from the start. The controller watches behavior, classifies failure shapes, and inserts one temporary nudge for the next decision only.

Source signal

Replit frames guidance as a runtime intervention, not a longer prompt. This page translates that idea into a small controller design.

Repo addition

The implementation makes guidance auditable with deterministic signals, evidence payloads, cooldowns, and a hard card budget.

Core claim

Static prompts degrade when every reminder is loaded up front. Guidance works better when it is situational, ephemeral, budgeted, and tied to evidence from the current run.

Controller questions

  • What is the agent doing right now?
  • Is this a known failure shape?
  • What is the smallest correction for the next turn only?

Best use cases

  • Repeated runtime errors or edit loops.
  • Unsafe commands too early in the run.
  • Mock-data escape and fake progress behavior.

Implementation shape

The Python engine models trajectory events, deterministic classifiers, scoring, cooldowns, and a short guidance-card bank. It intentionally keeps V1 logic inspectable.

Logic

Control loop mechanics

1. Observe trajectory

Collect messages, tool calls, failed checks, edits, and other runtime events instead of reasoning from prompt text alone.

2. Extract signals

Classify patterns such as repeated diagnostics, doom loops, unsafe changes, or fake-data escape with deterministic logic.

3. Select intervention

Rank candidate signals, apply a small card budget, enforce cooldowns, and pick only the most useful next-step guidance.

4. Discard after use

The intervention is turn-local. It is not promoted to a permanent hidden policy unless it proves to be a durable rule.

Verification

Experiment signals

Long debugging trace

The selector records multiple candidate failures but injects only the highest-priority cards so the prompt stays focused.

Cooldown handling

Repeated guidance is suppressed when the same signal keeps firing across nearby turns.

Risky cleanup scenario

Unsafe-change guidance fires when the agent reaches for destructive actions without enough evidence.
