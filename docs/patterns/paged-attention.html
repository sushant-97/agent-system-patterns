Paged Attention: OS-level memory for LLMs

Agent System Patterns · Deep read

SP
Sushant Pargaonkar June 2026 · 10 min read

Serving Large Language Models (LLMs) is incredibly memory-intensive. It's not just the model weights that consume VRAM; it's the context. The KV cache — which stores intermediate attention keys and values for every token — grows dynamically as the model generates new tokens. Before PagedAttention, managing this memory was highly inefficient. Memory fragmentation wasted up to 80% of VRAM, limiting batch sizes and throughput. PagedAttention fixed this by borrowing a classic idea from operating systems: virtual memory paging.

The Memory Fragmentation Problem

In autoregressive generation, you don't know in advance how many tokens a model will generate. The length of the output is unpredictable. Older LLM serving systems managed the KV cache by allocating contiguous blocks of memory for a sequence, guessing its maximum possible length.

This contiguous allocation leads to massive internal fragmentation (pre-allocated memory that is never used because the output was shorter than expected) and external fragmentation (available memory scattered in small chunks, unable to fit a new sequence).

The Block Table Solution

PagedAttention (introduced in vLLM) solves this by decoupling logical memory from physical memory. The KV cache is divided into fixed-size "blocks" (pages), where each block contains the keys and values for a fixed number of tokens (e.g., 16 or 32 tokens).

A Block Table manages the mapping between logical blocks of a sequence and physical blocks in GPU memory. When a sequence generates new tokens, it dynamically allocates a new physical block only when the current one is full. Physical blocks do not need to be contiguous in memory.

Unlocking Memory Sharing

Once you have page tables for memory management, another powerful capability unlocks: Memory Sharing. In many LLM workflows — like parallel decoding, beam search, or complex agent prompting — multiple sequences share the exact same system prompt or prefix.

With PagedAttention, the Block Table allows multiple sequences to point to the same physical blocks for their shared prefix. Memory is only allocated when they diverge (Copy-on-Write). This slashes memory usage for complex decoding tasks by up to 55%.

Conclusion

By treating the LLM KV cache like an operating system treats RAM, PagedAttention enables models to serve significantly larger batches of requests concurrently. It practically eliminates memory waste and turns complex sampling methods (like beam search) from memory-hogs into highly efficient operations.
